시스템 설계 명세서

여행 계획 공유 서비스

인터페이스 설계 (UI Flow)

사용자 인증부터 여행 계획 생성, 장소 추가 및 지도 확인까지의 핵심적인 사용자 흐름입니다.

Entity-Relationship Diagram (ERD)

- MEMBERS와 PLANS는 다대다(N:M) 관계이며, MEMBERS_PLANS_PIVOT 테이블로 연결됩니다.
- PLANS는 여러 개의 PLACES를 가질 수 있는 일대다(1:N) 관계입니다.
- MEMBERS는 여러 개의 PLACES를 생성할 수 있는 일대다(1:N) 관계입니다.

MEMBERS (사용자)

  • MEMBER_ID (PK)
  • SOCIAL_ID
  • NICKNAME
  • ...

PLANS (여행 계획)

  • PLAN_ID (PK)
  • TITLE
  • HOST_ID (FK to MEMBERS)
  • ...

MEMBERS_PLANS_PIVOT

  • MEMBER_ID (PK, FK)
  • PLAN_ID (PK, FK)
  • COLOR
  • ...

PLACES (장소)

  • PLACE_ID (PK)
  • PLAN_ID (FK to PLANS)
  • MEMBER_ID (FK to MEMBERS)
  • NAME
  • ...

Table: MEMBERS

사용자 정보를 관리하는 테이블

ERD

Column Data Type Constraints Description
MEMBER_ID NUMBER(19) PK, NOT NULL 사용자 고유 ID
SOCIAL_ID VARCHAR2(255) NOT NULL, UQ 소셜 로그인 ID
SOCIAL_TYPE VARCHAR2(20) NOT NULL, CHECK 소셜 로그인 타입 (GOOGLE, KAKAO...)
NICKNAME VARCHAR2(64) NOT NULL 닉네임
EMAIL VARCHAR2(255) NOT NULL, UQ 이메일
PROFILE_IMAGE VARCHAR2(255) 프로필 이미지 URL
CREATED_AT TIMESTAMP NOT NULL 생성일

Table: PLANS

여행 계획의 기본 정보를 관리하는 테이블

ERD

Column Data Type Constraints Description
PLAN_ID NUMBER(19) PK, NOT NULL 계획 고유 ID
TITLE VARCHAR2(255) NOT NULL 계획 제목
START_DATE DATE 여행 시작일
END_DATE DATE 여행 종료일
HOST_ID NUMBER(19) FK, NOT NULL 계획 생성자(Host) ID
SHARE_URI VARCHAR2(255) NOT NULL 공유용 고유 URI

Table: MEMBERS_PLANS_PIVOT

사용자와 계획의 N:M 관계를 정의하는 테이블

ERD

Column Data Type Constraints Description
MEMBER_ID NUMBER(19) PK, FK, NOT NULL 사용자 ID
PLAN_ID NUMBER(19) PK, FK, NOT NULL 계획 ID
COLOR VARCHAR2(16) NOT NULL 지도에 표시될 사용자별 색상
JOINED_AT TIMESTAMP NOT NULL 참여일

Table: PLACES

계획에 포함된 개별 장소를 관리하는 테이블

ERD

Column Data Type Constraints Description
PLACE_ID NUMBER(19) PK, NOT NULL 장소 고유 ID
MEMBER_ID NUMBER(19) FK, NOT NULL 장소 추가자 ID
PLAN_ID NUMBER(19) FK, NOT NULL 소속된 계획 ID
NAME VARCHAR2(255) 장소 이름
LATITUDE DECIMAL(9,6) NOT NULL 위도
LONGITUDE DECIMAL(9,6) NOT NULL 경도
TRAVEL_DATE DATE NOT NULL 방문 예정일
ORDER_IN_DAY NUMBER(10) NOT NULL 일자 내 방문 순서
